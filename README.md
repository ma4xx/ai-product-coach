# AI Product Coach

**一个让 AI 成为你产品陪练的 Skill——把模糊想法稳定转化为清晰、可执行的产品判断。**

适合：独立开发者、产品经理、vibe coder——任何需要和 AI 深挖产品方向、技术方案、商业决策的人。

---

## 它解决什么问题

你有没有遇到过这些情况：

- 脑海中只有一个模糊的感觉，不知道怎么跟 AI 说清楚
- AI 接到问题就给方案，跳过了你真正需要的追问
- 聊了很久，感觉深挖了，但产出的结论还是停在表面
- 不知道自己哪里想得不够深，也不知道下一步该怎么做

AI Product Coach 解决的就是这个问题。它让 AI 扮演**产品陪练**的角色，不管你带着多模糊的输入进来，都能被引导到「根因清晰、方案最小、可以直接执行」的产品判断。

---

## 它是怎么工作的

四步结构化框架，AI 主动引导，你只需要回答问题：

```
Step 0  命题提炼    ── 把模糊输入变成可追问的命题
Step 1  苏格拉底    ── 澄清边界、暴露假设
Step 2  5 Why       ── 往下凿，找到根因
Step 3  第一性原理  ── 从根因重建方案
Step 4  奥卡姆剃刀  ── 砍掉多余的，找到最小形态
```

跑完之后，AI 输出一页纸总结：问题 → 根因 → 方案 → 最小形态 → 下一步执行动作。

然后 AI 自动做复盘，评估这次产出的质量，并给出下一版迭代建议——**不需要你自己归纳，观察是 AI 的工作。**

## 快速开始

### 1. 安装 Skill

把 `SKILL.md` 放到你的 Claude Code 或 Codex 的 skills 目录：

```bash
# Claude Code
cp SKILL.md ~/.claude/skills/ai-product-coach/SKILL.md

# 或者直接 clone 整个仓库到 skills 目录
git clone https://github.com/YOUR_USERNAME/ai-product-coach ~/.claude/skills/ai-product-coach
```

### 2. 触发方式

在对话中说出以下任意一句，AI 就会调用这个 skill：

- 「深挖一下这个想法」
- 「帮我想清楚 ___」
- 「我有个想法，但不知道方向对不对」
- 「为什么我的产品 ___ 」
- 「我在 A 和 B 方案之间纠结」

或者直接丢一个模糊的想法进来，AI 会自动判断是否需要调用。

### 3. 典型使用场景

- **Vibe coding 前**：在动手写代码前，先深挖「这个功能到底解决什么问题」
- **产品决策**：面对多个方向不知道选哪个时
- **方案设计**：需要从根因出发重建方案，而不是在现有框架上打补丁
- **想法验证**：脑海中有一个感觉，想知道它有没有站得住脚

---

## 自我进化机制

每次跑完，AI 会自动输出：

1. **产出质量评估** ── 这次的根因可信吗？方案可执行吗？用户按这个去做会卡在哪？
2. **对你思维风格的观察** ── AI 观察你的表达习惯和思维盲区，随着使用次数增加越来越准
3. **迭代建议** ── 针对 skill 本身的具体改动建议，你可以选择立即更新或积累几次再改

**这意味着这个 skill 会随着你的使用越来越适应你的风格。**
